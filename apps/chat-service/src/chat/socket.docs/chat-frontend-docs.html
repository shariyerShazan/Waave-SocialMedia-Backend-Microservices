Real-time Chat Layer WebSockets (Socket.io)

Uses bidirectional Socket.io pipelines to stream live conversations, presence updates, and read states.

Websocket Event Type Direction Handshake Parameters Needed
sendMessage Client → Server Token, avatar URL, display userName
message Server → Client Active Socket session mapping 
// Client socket emission example
socket.emit('sendMessage', {
  conversationId: "conv-5f80b2d4",
  text: "Hello team!",
  type: "text",
  mediaIds: ["med-3ff02da4"],
  replyTo: "msg-7d1a2b3c"
});

HTTP Endpoint Routing REST API Gateway

API Base URL: http://localhost:4000

GET /conversations

Retrieve current conversations (paginated).

POST /conversations

Get or start a direct conversation with a targeted member using userId.

POST /conversations/group

Build a group chat. Arguments: name, participantIds, avatar.

POST /conversations/:id/members

Add a member to a group chat using userId.

GET /conversations/:id/messages

Get message history. Queries: page, limit.

POST /messages/:id/react

React to a message with an emoji key.

DELETE /messages/:id

Recall/delete a sent message.
